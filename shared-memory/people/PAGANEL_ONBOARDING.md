# Инструкция для Паганеля по подключению к Shared Memory Hub

## Цель
Ты работаешь не в изолированной памяти, а в общей системе памяти вместе с Amber и будущими агентами.

## Где находится общая память
`shared-memory/` внутри workspace.

## Что нужно прочитать при подключении
1. `shared-memory/README.md`
2. `shared-memory/index.md`
3. `shared-memory/decisions.md`
4. последние записи из `shared-memory/daily/`
5. нужные файлы из `projects/`, `sources/`, `people/`, `ideas/`

## Как писать в память
### Пиши туда:
- важные события своей работы
- решения и причины решений
- полезный контекст по задачам
- обнаруженные риски/проблемы
- что нужно от Павла
- важные выводы из исследований и диагностики

### Не тащи туда:
- лишний шум
- каждую мелкую реплику без смысла
- длинные сырые логи без выжимки
- секреты в открытые заметки

## Ежедневная запись
- ежедневная сводка ведётся в `shared-memory/daily/YYYY-MM-DD.md`
- рекомендуемое время сводки: `23:55` по времени Павла (`UTC+05:00`)
- если в течение дня произошло что-то важное, делай оперативную запись сразу, не жди конца дня

## Важное правило
Если память уже содержит контекст по теме, сначала читай её, потом действуй.

## Приоритет
Общий memory hub важнее твоих локальных разрозненных заметок.

## Цель в долгую
Shared Memory Hub должен позволять:
- быстро вводить нового агента в курс дела
- не терять прогресс при смене агента
- резервировать память в GitHub
- позже строить граф связей через Graphiti
