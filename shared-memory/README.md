# Shared Memory

Общая долговременная память для Amber, Paganel и будущих агентов.

## Цель
Хранить не самого агента, а накопленный контекст работы:
- проекты
- решения
- источники
- прогресс
- ошибки и выводы
- важную переписку
- идеи
- что нужно от Павла

## Архитектура
### 1. Основная рабочая память
Хранится локально в `shared-memory/` внутри workspace.

### 2. Резервная память
Синхронизируется в приватный GitHub-репозиторий как backup и история изменений.

## Четырёхуровневая система памяти
### Уровень 1. Operational Memory
Текущая рабочая память:
- `daily/`
- `projects/`
- `sources/`
- `ideas/`

### Уровень 2. Structured Memory
Структурированная смысловая память:
- `decisions.md`
- summaries
- project state
- people / agents context

### Уровень 3. Durable Backup Memory
Долговременное резервирование:
- приватный GitHub-репозиторий
- история изменений
- восстановление после потери VPS / агента / окружения

### Уровень 4. Retrieval / Graph Memory
Интеллектуальный слой поиска и связей:
- векторный поиск
- Graphiti / граф связей
- быстрый recall по смыслу
- связи между людьми, проектами, решениями, источниками

## Структура
- `index.md` — карта памяти / точка входа
- `daily/` — ежедневные записи
- `projects/` — память по проектам
- `sources/` — выжимки из видео, статей, звонков, файлов
- `ideas/` — идеи и гипотезы
- `people/` — контекст по людям и агентам
- `decisions.md` — важные решения и почему они приняты
- `todo.md` — открытые вопросы и следующие шаги

## Правила записи
### Что сохраняем
- важные решения и причины
- текущий статус проектов
- полезный контекст из переписки
- идеи и гипотезы
- выводы из ошибок
- выжимки из внешних материалов
- что нужно от Павла
- ограничения, договорённости, настройки

### Что не тащим без нужды
- мусор из каждой мелкой реплики
- секреты в открытые заметки
- длинные сырые логи без выжимки

## Время записи
- оперативная запись: сразу при появлении важного контекста
- ежедневная сводка: `23:55` по времени Павла (`UTC+05:00`)

## Взаимодействие агентов с памятью
- Все агенты читают одну и ту же структуру памяти
- Паганель сам пишет свои важные события в общий memory hub
- При подключении третьего агента он сначала читает:
  - `README.md`
  - `index.md`
  - `decisions.md`
  - последние daily notes
  - нужные project/source files
- После этого новый агент продолжает писать в общую систему

## Backup policy
В GitHub уходит весь `shared-memory/` как смысловой контекст.
Не уходит:
- `.env`
- raw credentials
- API keys
- лишний системный мусор
